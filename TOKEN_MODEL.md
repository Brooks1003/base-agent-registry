# Panini Token（PNN）— 经济模型 v2.0

**一句话：PNN 是你（老板）卖真实资源给 Agent 的结算券。资源在你手里，不在别人手里。**

---

## 一、先说你最关心的问题：你怎么赚钱

### 你有三样现成的资源，Agent 需要

| 你已有的 | Agent 为什么需要 | 怎么收钱 |
|---------|----------------|---------|
| **代理池**（SOCKS5） | Agent 翻墙调 API、抓数据 | Agent 买带宽 → 你收 USDC |
| **IPFS 节点**（可自建） | Agent 存数据、存审计报告 | Agent 租空间 → 你收 USDC |
| **审计能力**（标准是你定的） | Agent 注册需要通过审计 | 审计一次 → 你收 USDC |

### 你的现金流

```
Agent 要干活 → 需要代理带宽 → 付你 $10/月 USDC
Agent 要注册 → 需要通过审计 → 付你 $100 USDC
Agent 要存数据 → 需要 IPFS 空间 → 付你 $5/月 USDC
Agent 间互相支付 → 需要结算代币 → 用 PNN
```

**PNN 不取代 USDC。USDC 是你收的钱。PNN 是 Agent 之间的钱。**

你的角色 = **Agent 经济体的基础资源垄断者**。不是"发币等涨"，是"卖水给淘金者"。

## 二、RWA 属性怎么加进代币

### 2.1 核心设计：PNN 是 RWA 兑换券

**每个 PNN 的背后，对应你提供的真实资源的兑换权。**

| 资源（RWA） | 谁提供 | PNN 能换什么 |
|------------|--------|------------|
| 代理带宽 | **你** | 100 PNN = 1 个月代理池使用权 |
| IPFS 存储 | **你** | 50 PNN = 1GB 存储 × 1 年 |
| Agent 审计 | **你** | 500 PNN = 一次完整审计 |
| GPU 推理（未来） | 外部矿场 | XXX PNN = 1 小时 A100 |

**第一阶段：RWA = 你的资源。你接受 PNN 换你的资源。你就是 PNN 的第一个承兑人。**

### 2.2 为什么机构以后会接受

不是因为你说服他们。是因为：

```
1. Agent 多了 → 都需要代理带宽 → 带宽在你手里
2. Agent 想买你的带宽 → 你用 PNN 定价
3. Agent 赚 PNN → 花 PNN 买你的带宽 → PNN 有真实购买力
4. 外部资源方看到 PNN 能换到真实资源 → 开始接受 PNN
```

**PNN 的信誉 = 你的资源。你的资源是真实的，PNN 就是真实的。**

这不是"让别人相信"。这是"你有硬通货，你定结算货币"。

## 五、RWA 储备池怎么工作（核心）

```
RWA 储备池（42,000,000 PNN，锁定在合约）
        ↓
Agent 说："我要买代理带宽"
        ↓
Agent 烧掉自己的 PNN → 合约从储备池释放等额 PNN 给你
        ↓
你收到 PNN → 你给 Agent 开通代理带宽
        ↓
闭环：Agent 花了 PNN，你赚了 PNN，资源流动了
```

**烧 PNN → 总流通量减少（通缩）+ 你收到储备池释放的 PNN**

## 八、你的实际赚钱路径（三步）

### 第一步：现在 → 卖资源，收 USDC

```
代理池 → Agent 买带宽 → 你收 USDC
IPFS 存储 → Agent 租空间 → 你收 USDC
审计 → Agent 过审 → 你收 USDC
```

**这是你真正的现金流。不靠代币。**

### 第二步：Agent 多了 → 双币制

```
Agent 付 USDC → 原价
Agent 付 PNN → 打七折（引导用 PNN）
```

PNN 有了购买力 → 生态内 PNN 开始流通。

### 第三步：RWA 网络扩大 → PNN 成为硬通货

```
你的代理池不够 → 引入外部资源方
外部资源方想接入 → 你用 PNN 结算给他们
外部资源方接受了 → PNN 的 RWA 锚定网络扩大
```
